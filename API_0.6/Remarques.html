Mode de mise en œuvre et formats

Les API mises à disposition permettent d’interroger les données du référentiel transport de Tisséo.

Elles sont accessibles par l’appel de liens de type : http://pt.data.tisseo.fr/<Nom de l’api>?<paramètres>&key=<votre_clé>

Les API fournies sont :

API Description
placesList recherche de lieux
linesList liste des lignes
stopAreasList liste des zones d’arrêts
stopPointsList liste des poteaux d’arrêt
departureBoard horaires à un arrêt
messagesList liste des messages d’information
linesDisruptedList liste des lignes perturbées

Elles fournissent dans la mesure du possible une réponse selon 2 formats :

XML (défaut) et JSON.

CLES D’ACCES

L’utilisation des API est soumise à l’utilisation d’une clé attribuée à chaque demandeur. Cette clé doit être transmise lors de chaque appel.

Comment obtenir une clé ?

Pour obtenir une clé, il est nécessaire d’en faire la demande par mail à opendata@tisseo.fr en remplissant les champs suivants:

Champs Détails
vos références nom, prénom et/ou entreprise
votre mail (indispensable pour vous communiquer les changements et évolutions de l’API)
vos utilisations prévues (nom de projet ? objectifs ?)

COHERENCE DES ID AVEC LES FICHIERS OPENDATA

En plus de l’API temps réel, Tisséo fourni des fichiers descriptifs de son offre transport sur le portail OpenData de Toulouse Métropôle.

Ces fichiers sont disponibles aux formats standard Trident et GTFS sous la même licence ODBL [[LICENSE]] Voici les URL pour récupérer ces fichiers :

  • Format GTFS : http://data.toulouse-metropole.fr/les-donnees/-/opendata/card/16271-reseau-tisseo-metro-bus-tram-gtfs
  • Format Trident : http://data.toulouse-metropole.fr/les-donnees/-/opendata/card/14114-reseau-tisseo-metro-bus-tram-

Les fichiers sont mis à jour de manière automatique tous les lundis.

Afin d’étendre les possibilités d’utilisation de l’API, les identifiants GTFS et Trident sont maintenant identiques à ceux de l’API.

Ainsi par exemple, vous pourrez récupérer les tracés cartographiques des lignes dans le fichier GTFS et faire un affichage cartographique des lignes avec les informations temps réel de l’API.

Autre exemple, vous pouvez charger en cache dans vos applications toutes les horaires théorique via les fichiers et faire une correction temps réel de ces horaires avec l’API.

PERIMETRE DES DONNEES

  • Périmètre géographique :

Les données de l’API couvrent le perimètre de l’aire urbaine dans ça définition INSEE 2010 ainsi que la haute garonne.

  • Périmètre temporel :

Les données Transport Tisséo sont mises à jour quotidiennement voir en temps réel (horaires de passages bus et tram équipés de GPS).

  • L’Offre prévisionnelle :

L’horizon temporel des données est de 30 jours, mais nous préconisons un usage à 15 jours car l’offre à 30 jours peut changer avant application.

  • Date de péremption de la donnée :

Toutes les réponses de l’API contiennent un paramètre « expirationDate » qui indique jusqu’à quand l’information récupérée reste valable.

Il est fortement recommandé d’utiliser ce paramètre pour gérer la durée du cache de vos applications.

  • Réseau de transport disponibles :

A l’ouverture de l’API seul le réseau Tisséo sera disponible. Cependant, des paramètres « network » ont étés ajoutés en prévision d’une ouverture à d’autres réseau.

CREDIT OPENSTREETMAP

Les informations routières (noms de rues, numéros de rues, …) fournies par l’API TISSEO

proviennent du projet OpenStreetMap [[http://www.openstreetmap.org/]]

Certaines rues, ou certains numéros de rues ne sont pas encore connus d’OpenStreetMap et seront donc inaccessibles via notre API.

C’est pourquoi nous encourageons vivement les utilisateurs de l’API à contribuer à l’amélioration des données OpenStreetMap.

Nous réaliserons des mises à jour régulières qui contiendront vos contributions.

Techniquement, nous n’importons les numéros de rues que sur des nodes (addr:housenumber) et les nodes en relation à une rue (associatedStreet) et non pas sur les buildings.
